Privacy Policy

Last reviewed: July 18, 2026

Social Insights Studio provides analytics connection, synchronization, dashboard, and reporting tools for authorized social and website analytics sources.

Data we collect or process

  • Account and connection identifiers needed to operate authorized provider connections.
  • OAuth authorization records and encrypted provider tokens for connected sources.
  • Provider resource details such as selected TikTok accounts, Instagram professional accounts, Facebook Pages, YouTube channels, or Google Analytics 4 properties when those integrations are enabled.
  • Analytics metrics, content metadata, sync status, report metadata, and generated report artifacts needed to provide dashboards and exports.
  • Operational logs, errors, and security events needed to protect and maintain the service. Logs must not include provider secrets or access tokens.

How we use data

  • To authenticate provider connections authorized by the user.
  • To discover resources the user can access and create workspace-owned data sources for selected resources.
  • To synchronize permitted analytics, show dashboards, and generate reports requested by authorized workspace users.
  • To handle disconnect, deletion, troubleshooting, abuse prevention, reliability, and security operations.

Provider-specific notes

  • TikTok data is accessed through the approved TikTok scopes configured for the current connector.
  • Facebook Pages requests only pages_show_list, pages_read_engagement, and read_insights. Instagram via Facebook Login requests only instagram_basic, instagram_manage_insights, and the two Page discovery permissions pages_show_list and pages_read_engagement.
  • Meta data is limited to Pages and linked professional Instagram accounts the authorizing user explicitly selects. Social Insights Studio does not request publishing, comment-management, messaging, advertising, app-events, demographic, webhook, or broad business-management access.
  • YouTube uses only youtube.readonly to discover authorized channels and read channel/video metadata, and yt-analytics.readonly to read channel and video Analytics reports. Social Insights Studio does not request YouTube write, upload, advertising, messaging, or monetary-data access.
  • YouTube users are also subject to the YouTube Terms of Service.

Google API Services User Data Policy

Social Insights Studio's use and transfer of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements. Google user data is used only to provide and improve the user-facing analytics connection, synchronization, dashboard, support, security, and deletion features described here. It is not used for advertising, sold, or transferred for unrelated purposes.

Storage and retention

  • Provider credentials are stored server-side and outside the public web root.
  • Meta app-user and Page tokens are encrypted and retained while their authorization or selected resource is active, with a bounded 30-day deletion grace period after token expiry to allow reauthorization. Meta disconnect, deauthorization, data-deletion, and expired-authorization cleanup flows delete the associated credentials, resources, sync jobs, content, and stored insight snapshots. Raw Meta API responses are not retained by default.
  • Encrypted YouTube token records are retained only while the authorization is active. Disconnecting YouTube attempts Google revocation and immediately deletes credentials, discovered channels, and associated local snapshots, even if provider revocation fails.
  • Normalized channel, video, and Analytics snapshots are retained only for the active user-facing connection purpose. Social Insights Studio validates the authorization and refreshes stored YouTube data within each 30-day policy window; if an authorization cannot be validated by its deadline, the associated YouTube data is deleted. Raw Google or YouTube API responses are not stored by default.
  • Generated reports should have a short retention window and be deleted after expiry.

Sharing

Social Insights Studio does not sell provider data. Data may be shared only as needed to operate requested features, maintain security, comply with law, or work with infrastructure providers that support the service.

Your choices

  • You can disconnect supported provider connections from the product surface where available.
  • You can revoke Meta access from the relevant Facebook settings. A valid Meta deauthorization or data-deletion request triggers local deletion for the matching Meta authorization.
  • You can review or revoke Google access directly in Google Account third-party connections.
  • You can request deletion using the data deletion instructions.
